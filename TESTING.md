## 3. 运行方式

```bash
# 运行所有测试
npm test

# 运行指定测试文件
npm test -- --testPathPattern="tag-preset"
npm test -- --testPathPattern="default-tag"

# 运行带观察模式
npm run test:watch

# 按测试套件顺序运行（避免并发冲突）
npm test -- --runInBand

# 运行 lint
npm run lint
```

## 6. 手动验证要点

### 6.1 标签预设功能

1. 创建新预设，确认预设出现在列表中
2. 编辑预设名称/描述/颜色，确认更新生效
3. 添加标签到预设，确认标签项数量增加
4. 从预设移除标签，确认标签项数量减少
5. 删除预设，确认预设及其标签项被级联删除
6. 复制预设，确认新预设包含相同的标签项

### 6.2 默认预设功能

1. 选择一个预设，点击"设为默认"
2. 确认只有一个预设显示为默认
3. 确认默认预设卡片正确显示预设名称和标签预览
4. 点击"取消默认"，确认预设不再显示为默认
5. 确认页面刷新后默认状态保持

### 6.3 数据一致性

1. 设置默认预设后，关闭并重新打开应用，确认默认状态保持
2. 删除已成为默认的预设，确认自动取消默认状态
3. 在预设管理器中修改默认预设的标签，确认标签管理页面同步更新
