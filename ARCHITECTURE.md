# 架构说明

## 1. 分层边界

- `app/`：负责路由、页面容器与界面组合。
- `src/features/*`：按业务域拆分功能模块，如 `photo`、`tag`、`search`、`stats`、`backup`。
- `src/infra/db`：负责 SQLite 连接、数据库结构初始化与 repository 实现。
- `src/shared`：提供跨模块复用的类型、工具函数与基础 UI 组件。

## 2. 数据流

1. 页面触发用户操作。
2. `features/*` 中的 service 或 store 组织业务流程。
3. service 通过 `infra/db/repositories/*` 访问数据库。
4. repository 执行 SQL 并返回领域对象。
5. 页面根据最新状态渲染结果并反馈处理进度。

## 3. 数据库策略

- 应用启动时初始化当前数据库结构。
- 若检测到结构版本不匹配，则重建当前数据库结构并重新写入初始化数据。
- 分类与标签使用 `external_id` 作为备份导入导出时的稳定标识。
- 保留默认标签与扩展标签的初始化流程。

## 4. 关键设计

- 业务逻辑以显式 service 调用为主，避免隐式模块耦合。
- 备份导入接口统一支持：
  - `autoBackfill?`
  - `maxScanAssets?`
  - `onProgress?`
- 备份导入采用覆盖重建策略，并在导入后执行自动回填。
- 应用只保存照片引用与元数据，不复制系统原始照片文件。

## 5. 模块职责

- `photo`：照片导入、详情展示、标签绑定、资源修复。
- `tag`：分类与标签管理、默认标签设置。
- `search`：筛选条件构建、分页搜索与结果汇总。
- `stats`：聚合统计与时间分布展示。
- `backup`：导入导出、完整性校验、自动回填与进度反馈。

## 6. 当前非目标

- 云同步与多端实时协同。
- iOS/Web 与 Android 同等级交付质量。
